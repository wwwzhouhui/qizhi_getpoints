# OpenI 自动积分系统 - 功能测试指南

## 📋 已实现功能清单

### ✅ 1. 双重执行模式
- **手动执行**: `python3 login_getpoints.py`
- **自动定时**: `python3 login_getpoints.py --auto`
- **强制单次**: `python3 login_getpoints.py --once`
- **无头模式**: `python3 login_getpoints.py --headless`

### ✅ 2. 随机间隔调度
- 执行间隔：5、8、10、20、30、60、120、180、240、300分钟
- 最大间隔：不超过25小时
- 每次执行后动态计算下次执行时间

### ✅ 3. SQLite数据库状态管理
- **表结构**: user_records
  - id (INTEGER PRIMARY KEY AUTOINCREMENT)
  - username (TEXT)
  - runflag (INTEGER)
  - createtime (DATETIME)
  - isrun (INTEGER)
- **功能**: 记录每次执行状态，支持历史追踪

### ✅ 4. 双重日志系统
- **控制台输出**: INFO级别及以上（简洁）
- **文件输出**: DEBUG级别及以上（详细）
- **日志位置**: `logs/openi_automation_YYYYMMDD.log`
- **自动轮转**: 按日期自动创建新文件

### ✅ 5. 关闭按钮自动化
- **XPath**: `/html/body/div[2]/div[2]/div[2]/div/div[2]/div[2]/div[1]/div[4]/div[2]/table/tbody/tr[1]/td[12]/div/div/div/div/a[2]`
- **功能**: 任务完成后自动点击关闭按钮
- **回退**: JavaScript强制点击

## 📝 使用流程

### 首次使用
1. 编辑 `config_getpoints.ini`，填入账号信息
2. 运行测试: `python3 login_getpoints.py --headless`
3. 检查日志: `cat logs/openi_automation_*.log`
4. 验证数据库: `python3 -c "..."`

### 启用自动模式
1. 修改配置: `auto_execute_enabled = true`
2. 启动服务: `python3 login_getpoints.py --auto`
3. 查看调度: 日志显示"下次执行时间"
4. 优雅退出: 按 `Ctrl+C`

### 监控运行状态
```bash
# 查看执行记录
python3 -c "
import sqlite3
conn = sqlite3.connect('user_records.db')
cursor = conn.cursor()
cursor.execute('SELECT COUNT(*) FROM user_records WHERE DATE(createtime)=DATE(\"now\")')
today_count = cursor.fetchone()[0]
print(f'今日执行次数: {today_count}')
conn.close()
"

# 查看实时日志
tail -f logs/openi_automation_$(date +%Y%m%d).log

# 检查调度器状态
ps aux | grep login_getpoints.py
```

## ⚠️ 注意事项

1. **数据库文件**: `user_records.db` 包含所有运行记录，请定期备份
2. **日志管理**: 日志文件不会自动清理，需要手动删除旧文件
3. **随机间隔**: 每次执行后重新计算，范围5分钟-25小时
4. **优雅退出**: 使用 `Ctrl+C` 停止，避免强制杀死进程
5. **浏览器保持**: 执行完成后浏览器会保持1小时，可通过修改代码调整

## 🐛 故障排除

### 问题1: 数据库ID显示0
**原因**: 使用了跨连接的 `last_insert_rowid()`
**解决**: ✅ 已修复，使用 `cursor.lastrowid`

### 问题2: 日志文件无法创建
**原因**: 目录不存在或权限问题
**解决**: 检查 `logs/` 目录权限，确保可写

### 问题3: 调度器不执行
**检查**: 确认 `auto_execute_enabled = true`
**验证**: 查看日志中的"定时调度器已启动"

### 问题4: 关闭按钮点击失败
**原因**: XPath可能已变化
**解决**: 检查页面元素，更新XPath配置
